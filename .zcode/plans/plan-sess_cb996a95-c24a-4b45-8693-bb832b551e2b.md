## Part A — New backend (apps/api)

### A1. Notification module (build from scratch)
Mirror the `request` module pattern exactly.

- **`apps/api/src/typeorm/entities/notification.entity.ts`** — new entity `Notification`:
  - Fields: `id` (uuid), `userId` (recipient, nullable for broadcast), `centerId` (nullable), `title`, `message`, `type` (enum: `BOOKING|PAYMENT|DEPOSIT|LEAD|SYSTEM|REQUEST|EVENT`), `priority` (enum: `LOW|MEDIUM|HIGH`, default MEDIUM), `read` (bool, default false), `actionUrl` (nullable, deep-link target), `metadata` (jsonb nullable), `createdAt`, `updatedAt`.
  - Relations: `ManyToOne` User (recipient), `ManyToOne` Center.
- **`apps/api/src/graphql/inputs/notification.input.ts`** — `CreateNotificationInput`, `NotificationFiltersInput`, `NotificationStatistics` object type.
- **`apps/api/src/graphql/resolvers/notification.resolver.ts`** — `NotificationResolver`:
  - Queries: `notifications(filters?)` (filter by userId/read/type/centerId, paginated, newest first), `notification(id)`, `notificationStats(userId?, centerId?)` (total/unread/byType), `myNotifications` (current user from context).
  - Mutations: `createNotification(input)`, `markNotificationRead(id)`, `markAllNotificationsRead(userId?)`, `deleteNotification(id)`, `sendNotification(input)` (broadcast/multi-recipient create used by the Send dialog).
- **`apps/api/src/notification/notification.module.ts`** — `TypeOrmModule.forFeature([Notification])` + `CacheModule` + resolver. Export resolver.
- **`apps/api/src/typeorm/typeorm.module.ts`** — add `Notification` to `ALL_ENTITIES` + imports.
- **`apps/api/src/app/app.module.ts`** — import + register `NotificationModule`.

### A2. Center settings persistence (for Settings pages)
Add to the existing `apps/api/src/graphql/resolvers/center.resolver.ts`:
- Query `centerSettings(centerId)` → returns the `settings` jsonb (or empty default).
- Mutation `updateCenterSettings(centerId, settings)` → deep-merges partial settings into `Center.settings` and returns the updated object. (Reuses existing entity + repo; no schema/entity change.)

### A3. Seed demo notifications
Extend `seed.js` to insert 4–6 sample notifications so the feed is non-empty in dev.
